Here's how you can convert your Python unittest test class into a Java JUnit test case:

```java
import static org.junit.Assert.assertEquals;
import org.junit.Test;

public class Tester {

    @Test
    public void testCalculateEuclideanDistance() {
        // Test data
        double[] point1 = {1.0, 2.0};
        double[] point2 = {4.0, 6.0};

        // Expected result
        double expectedResult = 5.0;

        // Actual result
        double actualResult = calculateEuclideanDistance(point1, point2);

        // Asserting the expected and actual results
        assertEquals(expectedResult, actualResult, 0.0);
    }

    private double calculateEuclideanDistance(double[] point1, double[] point2) {
        return Math.sqrt(Math.pow(point2[0] - point1[0], 2) + Math.pow(point2[1] - point1[1], 2));
    }
}
```

In this Java code:

- We have imported necessary packages (`static org.junit.Assert.assertEquals` for assertion and `org.junit.Test` for marking the method as a test case).
- Created a Java class named `Tester`.
- Inside the `Tester` class, created a method `testCalculateEuclideanDistance()` annotated with `@Test`, which will be executed as a unit test.
- Defined test data and expected result similar to your Python test.
- Calculated the actual result using the `calculateEuclideanDistance` method.
- Used `assertEquals` to compare the expected result with the actual result. The third argument `0.0` specifies the delta for floating-point comparison, allowing for some margin of error.
- Implemented the `calculateEuclideanDistance` method inside the same class, converting it from Python to Java syntax.
